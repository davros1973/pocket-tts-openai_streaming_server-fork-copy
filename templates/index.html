Endpoints
MethodPathDescription
GET/healthHealth check → {"status":"healthy","model_loaded":true,...}
GET/v1/voicesList all available voices (OpenAI-compatible format)
POST /v1/audio/speech Generate speech — OpenAI TTS API compatible.
Body: model (ignored), input (text), voice (ID), response_format (wav/pcm/mp3), stream (bool).
Returns streaming WAV/PCM or complete MP3 file.
cURL Examples
curl -X POST http://localhost:8884/v1/audio/speech \
  -H "Content-Type: application/json" \
  -d '{"model":"pocket-tts","input":"Hello world","voice":"alba","response_format":"mp3","stream":false}' \
  --output speech.mp3
curl -X POST http://localhost:8884/v1/audio/speech \
  -H "Content-Type: application/json" \
  -d '{"model":"pocket-tts","input":"Hello world","voice":"alba","response_format":"wav","stream":true}' \
  --output speech_stream.wav
LiteLLM Integration
This service is OpenAI TTS-compatible — add it to LiteLLM's config.yaml (TTS proxy), or call it directly from OpenWebUI / any OpenAI TTS client by pointing api_base at this service. 
model_list:
  - model_name: pocket-tts
    litellm_params:
      model: openai/tts-1
      api_base: http://192.168.70.171:8884
      api_key: dummy
      stream: true
Python / OpenAI SDK
from openai import OpenAI

client = OpenAI(
    api_key="dummy",
    base_url="http://localhost:8884"
)

with client.audio.speech.with_streaming_response.create(
    model="pocket-tts",
    voice="alba",
    input="Hello from Pocket TTS running on xarta!"
) as resp:
    resp.stream_to_file("speech.mp3")

Setup: Voice Cloning Model
1
Accept gated model terms
The full Pocket TTS model (kyutai/pocket-tts) with voice cloning is a gated HuggingFace repository. Visit huggingface.co/kyutai/pocket-tts and accept the usage terms while logged in to your HuggingFace account.
2
Add your HuggingFace token
Edit the stack's .env file and set:
HF_TOKEN=hf_your_token_here
Get a token at huggingface.co/settings/tokens (needs read access on the gated repo).
3
Restart the container
Run bash deploy.sh --up from the stack directory, or restart via Dockge. The model will download from HuggingFace on first start (~800 MB).
4
Upload voice samples below
Record a clear 10–30 second WAV clip (mono or stereo, 16 kHz+ recommended). Avoid background noise. Uploaded voices are stored in voices_user/ which is .gitignored — they will never be committed to the repo.

Streaming Info
Streaming endpoint: POST /v1/audio/speech with "stream": true

When streaming is enabled, the response is a chunked Transfer-Encoding HTTP stream in WAV or raw PCM format. The model generates audio incrementally — the first chunk arrives within ~200ms and the response continues until synthesis is complete.

Monitor note: This service does not expose a /v1/monitor/stream SSE endpoint. Use container logs for real-time inference output:
pct exec 841 -- docker logs -f pockettts-openai-tts
Batch Generation (stream: false — returns complete file)
Real-time Streaming (stream: true — OpenAI API, Web Audio API)
How this works: Sends POST /v1/audio/speech with "stream": true, "response_format": "wav". Reads the chunked HTTP response body using ReadableStream. Skips the 44-byte WAV header, then decodes each arriving network chunk as int16 LE PCM → float32 and schedules it on the Web Audio API AudioContext. First audio plays within ~200ms.
